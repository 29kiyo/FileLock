# FileLock

Windows用のファイル・フォルダーロックアプリ（C# / WPF / .NET 8）。

## 仕組みの概要

- ロックは **NTFSのアクセス許可（ACL）に「拒否」ルールを追加**することで実現しています。
  この方式のため、**アプリが起動していなくても** OS レベルでロックが有効です。
- zip/rar/7z などの圧縮ファイルも「ファイル」として同じ仕組みでロックされるため、
  ロックされたアーカイブを解凍しようとすると、まずアプリでロックを解除（パスワード入力）
  しない限り、Explorer・7-Zip・WinRAR などから開けません。
- グループの「一時アンロック」は、実際にACLの拒否ルールを外し、Windowsタスクスケジューラに
  再ロック用のタスクを1回だけ登録することで、アプリを閉じていても指定時間後に自動で
  再ロックされるようにしています。
- アンインストール時は、Inno Setup の `[UninstallRun]` により、ファイル削除前に
  `FileLock.exe --unlock-all` が実行され、**パスワード不要で全ロックを解除**してから
  アンインストールが進みます（パスワードを忘れた場合の保険）。

## ⚠️ 重要な制限事項（正直な説明）

NTFSのACLによる拒否ルールは、Windowsの標準的なアクセス制御であり、**暗号化ではありません**。
ファイルの所有者（＝あなた自身のアカウント）は、理論上いつでもエクスプローラーの
「プロパティ→セキュリティ→編集」や `icacls` コマンドから権限を直接変更してロックを
解除できてしまいます。これは Windows の仕様上、所有者から権限変更能力を完全に奪うことが
できないためです。

このツールは「うっかり削除・閲覧してしまうことの防止」や「家族・同僚など詳しくない人の
簡易的なアクセス防止」には有効ですが、**セキュリティ専門家からの防御や、Windowsの操作に
詳しい人からの防御を保証するものではありません**。機密性の高いデータを本当に守りたい
場合は、BitLockerやVeraCryptなどの暗号化ツールの利用を検討してください。

なお、自分のアカウントが所有するファイルへのACL変更に管理者権限は不要なため、
本アプリは**管理者権限なし（UACプロンプトなし）**で動作します。インストーラーも
ユーザーごとのインストール（`{userpf}`配下）になっており、管理者権限は要求しません。

## プロジェクト構成

```
FileLock/
├─ src/                     WPFアプリ本体 (FileLock.csproj)
│  ├─ Models/                データモデル（グループ、設定、テンプレート）
│  ├─ Services/               ロック処理(ACL)・パスワード・保存・スケジューラ
│  └─ Views/                  パスワード入力・グループ編集・ファイル追加・一時アンロックの各画面
├─ installer/setup.iss      Inno Setup インストーラースクリプト
└─ .github/workflows/build.yml  GitHub Actions ビルド・リリースワークフロー
```

## GitHub Actions ワークフローの使い方

1. GitHubリポジトリの「Actions」タブ → 「Build and Release」を選択
2. 「Run workflow」をクリック
3. `version` にリリースしたいタグ（例: `v1.0.0`）を入力して実行
4. 完了すると、指定したタグ名でGitHub Releaseが自動作成され、
   `FileLock-Setup-<version>.exe`（インストーラー）が添付されます（アーティファクトへの
   アップロードは行わず、直接リリースを作成します）

ワークフローは Windows ランナー上で以下を行います:
1. `dotnet publish` で自己完結の単一exe (`publish/FileLock.exe`) を生成
2. Chocolatey で Inno Setup をインストールし、`installer/setup.iss` をビルド
   （バージョン文字列は `/DMyAppVersion=...` で埋め込み）
3. `softprops/action-gh-release` で、そのインストーラーexeを直接GitHub Releaseに公開

## ローカルでのビルド（Windows環境）

```powershell
dotnet publish src/FileLock.csproj -c Release -r win-x64 --self-contained true -p:PublishSingleFile=true -o publish
```

Inno Setup がインストールされていれば、`installer/setup.iss` を ISCC.exe でコンパイルすると
インストーラーexeが `installer/Output/` に生成されます。

## 主な機能と対応方法

| 要件 | 対応方法 |
|---|---|
| アンインストールでロック解除（パスワード不要） | `[UninstallRun]` が `--unlock-all` を実行 |
| アプリ未起動でもロック有効 | NTFS ACLの拒否ルールを直接ファイルに設定（OSレベル） |
| 管理GUIはパスワード不要 | 起動時は即メイン画面を表示。編集操作のみパスワードを要求 |
| グループ編集はパスワード必須、ウィンドウ操作中は再度聞かない | `PasswordGraceMinutes`（分）でグループごとに設定可能。**0の場合は管理画面を閉じる／アプリを終了するまで再入力不要**、1以上ならその分数で自動的に再度パスワードを要求 |
| パスワード2回入力＋目のアイコンで表示切替 | `GroupEditWindow` / `PasswordPromptWindow` に実装 |
| ファイルパスをドラッグ＆ドロップで別グループへ移動 | `MainWindow` のTreeViewでドラッグ＆ドロップ、移動元グループのパスワードを要求 |
| zip/rar/7z解凍前にパスワードを要求 | アーカイブファイル自体をロック対象にすることで、解凍ツールがファイルを開けない状態にする |
| グループごとの一時アンロック（N時間パスワード不要） | `TempUnlockWindow` でACLを解除し、タスクスケジューラで期限後に自動再ロック |
| テンプレートのエクスポート／インポート | `.flt`（JSON）ファイルとして、パスワード・ファイル一覧を除く設定のみを保存/読込 |
